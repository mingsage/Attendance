1. 后端 FastAPI 项目结构搭建。
2. 前端 Vue 3 + Vite + Element Plus 项目结构搭建。
3. SQLite 数据库、ORM 模型和基础表结构。
4. 登录认证、JWT、密码哈希。
5. 前后端接口封装和路由守卫。
6. 基础页面框架、侧边栏、登录页、总览页。

## 三、关键代码说明

### 3.1 后端入口

文件：`backend/app/main.py`

作用：

- 创建 FastAPI 应用。
- 注册 CORS，允许前端访问。
- 挂载 `/uploads` 和 `/face_db` 静态资源。
- 启动时创建数据库表。
- 首次运行时创建默认教师账号 `teacher / 123456`。

验收讲法：

> 后端入口负责把所有 API 模块挂到 FastAPI 上，同时初始化数据库和静态文件路径。这样前端既能调用接口，也能访问上传后的图片资源。

### 3.2 API 路由注册

文件：`backend/app/api/__init__.py`

模块划分：

| 模块 | 作用 |
|---|---|
| `auth` | 登录、注册、改密 |
| `students` | 学生管理、人脸库导入 |
| `attendance` | 考勤签到、活体动作、考勤记录 |
| `group_photo` | 合照识别 |
| `statistics` | 统计数据和导出 |
| `emotions` | 情绪明细 |

验收讲法：

> 我们按业务域拆路由，每个模块只负责一类功能，后期维护时比较清晰。

### 3.3 数据模型

主要文件：

| 文件 | 表 | 说明 |
|---|---|---|
| `backend/app/models/user.py` | `users` | 登录账号，含角色和学生绑定 |
| `backend/app/models/student.py` | `students` | 学生基础信息和人脸特征 |
| `backend/app/models/attendance.py` | `attendance_records` | 考勤记录 |
| `backend/app/models/emotion.py` | `emotion_records` | 情绪记录 |
| `backend/app/models/activity.py` | `activity_participation` | 合照活动参与记录 |

验收讲法：

> 数据库围绕学生展开，用户账号通过 `student_id` 绑定学生，考勤、情绪和活动参与也都通过 `student_id` 关联到学生。

### 3.4 登录与权限

主要文件：

- `backend/app/core/security.py`
- `backend/app/api/auth.py`
- `backend/app/api/deps.py`
- `frontend/src/stores/auth.js`
- `frontend/src/router/index.js`

实现方式：

1. 登录成功后后端生成 JWT。
2. 前端保存 token、用户名、角色。
3. 请求接口时 Axios 自动带上 `Authorization`。
4. 后端用 `get_current_user` 解析 token。
5. 教师接口用 `require_teacher` 限制。

验收讲法：

> 前端负责页面入口控制，后端负责最终安全校验。密码通过 bcrypt 哈希保存，不存明文。

## 五、老师可能问的问题

### Q1：为什么说这是 BS 架构？

答：浏览器端用 Vue 负责页面和交互，服务端用 FastAPI 提供接口和处理识别逻辑。浏览器不直接访问数据库，而是通过 HTTP API 和后端通信，所以是典型 Browser/Server 架构。

### Q2：为什么用 FastAPI？

答：FastAPI 写接口比较清晰，支持 Pydantic 参数校验，也能自动生成接口文档，适合这种前后端分离的实验项目。

### Q3：数据库为什么用 SQLite？

答：SQLite 部署简单，不需要额外数据库服务，适合课程设计和单机演示。代码里使用 SQLAlchemy，如果以后要换 MySQL，也只需要调整连接配置和少量兼容点。

### Q4：密码怎么保证安全？

答：密码不直接保存，后端使用 bcrypt 生成哈希。登录时用 `verify_password` 比对输入密码和哈希值。

### Q5：前端隐藏菜单就安全吗？

答：不够。所以我们后端也加了权限依赖，比如教师接口必须通过 `require_teacher`，学生直接请求也会被拒绝。
