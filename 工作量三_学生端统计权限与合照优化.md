# 工作量三：学生端、统计权限与合照优化

> 定位：围绕“数据可看、权限正确、合照和情绪统计更完整”的优化工作。  
> 这部分适合明天验收时补充说明系统的完整性和安全边界。

---

## 一、工作范围

本工作量主要负责：

1. 教师端学生管理优化。
2. 批量导入人脸时自动创建学生账号。
3. 教师重置学生密码。
4. 学生端“我的人脸”和“我的账户”。
5. 学生只能查看自己的考勤和情绪数据。
6. 合照识别和活动参与统计。
7. 情绪统计图和明细表。
8. 考勤统计、活动统计、Excel 导出相关优化。

## 三、关键代码说明

### 3.1 学生管理和账号自动创建

主要文件：

- `frontend/src/views/Students.vue`
- `backend/app/api/students.py`
- `backend/app/models/student.py`
- `backend/app/models/user.py`

实现点：

| 功能 | 说明 |
|---|---|
| 新增学生 | 创建 `Student` 后同步创建 `User` |
| 批量导入人脸 | 文件名解析：学号、姓名、班级、性别 |
| 默认学生密码 | `123456` |
| 重置密码 | 教师可在学生管理页面重置 |
| 多张人脸样本 | 使用平均特征合并，提升稳定性 |
| 删除学生 | 同时删除考勤、情绪、活动参与、账号和人脸照片 |

答辩口径：

> 教师只需要批量导入带有规范命名的人脸照片，系统就能自动创建学生基础信息和登录账号，降低录入成本。

### 3.2 学生端我的人脸和账户

主要文件：

- `frontend/src/views/MyProfile.vue`
- `backend/app/api/students.py`
- `backend/app/api/auth.py`

实现点：

| 功能 | 说明 |
|---|---|
| 查看学生信息 | 学号、姓名、班级、性别、人脸状态 |
| 查看当前数据库照片 | 点击当前人脸照片可放大查看 |
| 上传/拍照录入人脸 | 学生可以维护自己的人脸照片 |
| 修改密码 | 学生可在“我的账户”中修改密码 |

答辩口径：

> 学生端不开放管理能力，只能维护自己的账号和人脸信息；学生身份通过 token 绑定到 `student_id`，后端按这个 ID 查自己的信息。

### 3.3 合照识别

主要文件：

- `frontend/src/views/GroupPhoto.vue`
- `backend/app/api/group_photo.py`
- `backend/app/models/activity.py`

后端流程：

```text
教师上传合照
  -> 检测所有人脸
  -> 对每张人脸提取特征
  -> 与学生底库逐一比对
  -> 去重，防止同一学生重复计入
  -> 记录 ActivityParticipation
  -> 同步记录 EmotionRecord，source=group_photo
  -> 返回识别名单
```

答辩口径：

> 合照识别默认不做活体检测，因为合照本身是活动记录场景，不是实时身份认证场景。我们重点做多人脸检测、匹配、去重和活动参与统计。

### 3.4 情绪统计

主要文件：

- `frontend/src/views/EmotionStats.vue`
- `backend/app/api/emotions.py`
- `backend/app/api/statistics.py`
- `backend/app/models/emotion.py`

实现点：

| 角色 | 可见数据 |
|---|---|
| 教师 | 全部学生的考勤和合照情绪数据 |
| 学生 | 仅自己的考勤情绪数据 |

最近修复：

- 学生登录后情绪统计不再看到全班数据。
- 学生只看 `student_id=自己` 且 `source=attendance` 的情绪记录。
- 合照情绪默认作为教师统计和活动分析的一部分，不混入学生个人考勤情绪。

答辩口径：

> 情绪记录表里用 `source` 区分来源，考勤是 `attendance`，合照是 `group_photo`。学生端只展示自己的考勤情绪，教师端可以看全局。

### 3.5 统计和导出

主要文件：

- `backend/app/api/statistics.py`
- `backend/app/services/export_service.py`
- `frontend/src/views/Records.vue`
- `frontend/src/views/Dashboard.vue`
- `frontend/src/views/EmotionStats.vue`

实现点：

| 功能 | 说明 |
|---|---|
| 考勤统计 | 按课程和日期统计应到、实到、缺勤、到课率 |
| 情绪统计 | ECharts 饼图展示情绪分布 |
| 活动统计 | 基于合照识别记录统计参与次数 |
| Excel 导出 | 导出考勤明细和统计报表 |

答辩口径：

> 统计数据不是前端自己算，而是后端从数据库聚合后返回。导出功能也在后端生成 xlsx，前端只负责下载。

## 五、老师可能问的问题

### Q1：批量导入人脸怎么知道学生信息？

答：通过文件名解析，例如 `学号-姓名-班级-性别.jpg`。后端会提取文件名中的字段，如果学生不存在就创建学生，并同步创建登录账号。

### Q2：学生默认密码是多少？

答：默认密码是 `123456`。密码入库前会用 bcrypt 哈希。教师也可以在学生管理里把学生密码重置回 `123456`。

### Q3：学生为什么看不到别人的情绪统计？

答：后端查询时检查当前用户角色。如果是学生，就加上 `student_id == 当前学生ID`，并且只查询 `source=attendance` 的考勤情绪。

### Q4：合照识别为什么不做活体检测？

答：合照识别是活动记录场景，输入本身是一张活动照片，目标是识别照片中有哪些学生参加了活动。活体检测主要用于实时签到防伪，所以放在考勤签到流程里。

### Q5：统计报表是怎么来的？

答：后端基于 `AttendanceRecord`、`EmotionRecord`、`ActivityParticipation` 聚合数据，再返回给前端图表或生成 Excel。
