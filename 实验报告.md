## 一、实验目标

设计并实现一套基于 B/S 架构的班级考勤系统，支持以下核心功能：

- **人脸识别签到**：通过摄像头采集人脸，与人脸库进行比对识别
- **活体检测**：基于图像质量评估和随机动作提示，防范照片/视频伪造签到
- **合照识别**：上传活动合影，批量检测并识别人脸，自动生成参与名单
- **情绪分析**：签到时同步分析情绪状态，提供统计图表
- **考勤管理**：记录查询、筛选、统计报表与 Excel 导出

## 五、核心功能实现

### 5.1 人脸识别流程

```
摄像头采集 → 图像质量检查（清晰度/亮度） → 人脸检测（OpenCV Haar Cascade）
→ 特征提取 → 与人脸库特征计算余弦相似度 → Top-1 匹配（阈值 0.58）
→ 写入考勤记录 + 情绪分析
```

### 5.2 活体检测

- **静默活体**：分析图像清晰度（拉普拉斯方差）、亮度分布、纹理复杂度，综合评分
- **动作活体**：签到前随机展示动作提示（眨眼、张嘴、摇头、点头），降低照片/录屏攻击风险

### 5.3 合照识别

```
上传合照 → OpenCV 批量人脸检测 → 逐人脸提取特征 → 与人脸库比对
→ 写入 activity_participation 表 → 前端展示柱状图统计
```

### 5.4 模型扩展预留

因课堂环境要求可直接运行，当前使用轻量实现。服务层均定义了接口，替换真实模型时不需修改 API：

| 服务 | 当前实现 | 可替换为 |
|------|----------|----------|
| 人脸检测/特征 | OpenCV + 轻量特征向量 | insightface（RetinaFace + 512 维特征） |
| 情绪分析 | 启发式图像特征分类 | DeepFace / FER |
| 活体检测 | 图像质量评估 + 动作提示 | EAR 眨眼检测 / 多帧光流分析 |

## 六、测试步骤

1. **启动后端**：`uvicorn app.main:app --reload`，访问 `/docs` 确认 API 可用
2. **启动前端**：`npm run dev`，浏览器打开 `http://127.0.0.1:5173`
3. **登录系统**：使用 `teacher / 123456` 登录
4. **录入学生**：新增学生 → 上传人脸照片 → 确认"人脸库"状态变为"已录入"
5. **考勤签到**：使用摄像头拍照签到 / 自动倒计时捕获 / 上传图片签到
6. **合照识别**：上传活动合影 → 查看识别结果
7. **查看统计**：总览仪表盘、情绪统计、活动参与统计
8. **导出数据**：筛选考勤记录 → 导出 Excel → 检查文件内容

## 八、总结与改进方向

系统实现了考勤实验要求的主要功能，覆盖人脸签到、活体检测、情绪分析、合照识别和数据导出等核心模块。服务层通过接口封装保留了替换 insightface、DeepFace 等工业级模型的能力。

**改进方向**：

- 使用 RetinaFace / MTCNN 替代 Haar Cascade 提升检测精度
- 接入 insightface 的 ArcFace 特征提取提高识别准确率
- 引入多帧动作活体（眨眼检测、头部姿态估计）增强防伪能力
- 使用 Redis 缓存人脸特征，提升大规模班级的签到响应速度
- 替换 SQLite 为 MySQL / PostgreSQL 以支持并发读写
