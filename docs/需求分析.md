# 班级考勤系统 — 需求分析

> 基于 `实验任务` 原始要求 + `request.md` 评分标准的综合需求分析

---

## 一、项目背景与目标

### 1.1 原始实验任务

> 以**内容安全领域的人脸识别、活体检测技术**为核心，结合 BS（Browser/Server）架构开发，实现一套功能完整、安全可靠的班级考勤系统。要求融入合照学生识别、学生面部情绪分析功能。

关键词解读：

| 关键词 | 含义 | 对应实现 |
|---|---|---|
| 内容安全 | 防止虚假考勤（照片/视频伪造），确保签到数据真实 | 活体检测是核心防线 |
| 人脸识别 | 检测+比对，确认签到者身份 | 检测(SCRFD) + 识别(ArcFace) |
| 活体检测 | 区分真人与照片/屏幕/视频 | 11 维综合评分方案 |
| BS 架构 | 浏览器端操作，服务端处理 | Vue 3 + FastAPI |
| 合照识别 | 批量检测+识别，生成参与名单 | 合照上传→SCRFD 全检测→逐一比对 |
| 情绪分析 | 同步分析面部表情，分类统计 | HSEmotion + DeepFace 混合 |

## 二、功能需求

### F1 基础考勤功能（25分）

**原始要求**：
> 前端通过浏览器调用摄像头采集学生人脸信息，传输至后端；后端完成活体检测、人脸库比对，将考勤结果返回至前端渲染显示。

**数据流**：

```
浏览器摄像头 ──→ canvas 拍照 ──→ HTTP 上传 ──→ 后端
                                                    │
  前端渲染 ←── JSON 响应 ←── 判定 ←── 活体检测(11维)
    │                                        │
    ├ 成功：学号/姓名/班级/置信度/情绪/活体分    └── 人脸比对(512d 余弦)
    └ 失败：具体原因(活体不过/未匹配/动作未完成)
```

**需求分解**：

| 编号 | 需求 | 分值 | 说明 |
|---|---|---|---|
| F1.1 | 摄像头调用正常、人脸采集流畅 | 3 | `getUserMedia` API，720p，支持 Chrome/Firefox/Edge |
| F1.2 | 支持手动/自动捕捉并显示采集状态 | 3 | 手动拍照 + 3 秒倒计时 + 连续自动捕捉三模式 |
| F1.3 | 实时渲染后端返回的学生信息 | 3 | 学号/姓名/性别/班级/置信度/情绪/活体分/时间 |
| F1.4 | 支持筛选查询考勤记录 | 3 | 按姓名/学号关键词 + 成功/失败状态筛选，分页排序 |
| F1.5 | **活体检测可抵御照片、视频欺骗** | **10** | 核心得分项：多维度防欺骗，11 维综合评分 |
| F1.6 | 班级人脸库批量导入、增删改 | 2 | 文件名解析(学号-姓名-班级-性别)，单张/批量上传 |
| F1.7 | 记录考勤数据并 Excel 导出 | 1 | 全量导出 + 课程维度统计导出 |

**F1.5 活体检测详细要求**（10 分核心项）：

| 检测维度 | 权重 | 对抗目标 |
|---|---|---|
| DeepFace MiniFASNet 深度学习 | 40% | 通用照片/视频翻拍 |
| rPPG 脉搏波（绿色通道 FFT） | 15% | **照片/视频无心跳信号** |
| 光流时序一致性 | 12% | **手持照片晃动（刚体平移）** |
| 摩尔纹专项 2D-FFT | 10% | **屏幕翻拍干涉条纹** |
| FFT 频域分析 | 5% | 通用频谱衰减异常 |
| LBP 纹理分析 | 5% | 纸张/屏幕纹理异常 |
| 色彩/清晰度/亮度/边缘 | 各 2-3% | 辅助判断 |
| 动态阈值 | — | 自适应光照变化 |

**动作挑战机制**：4 种随机动作（微笑/左转/右转/张嘴），15 秒过期，后端强制验证。rPPG < 0.15 一票否决。

### F2 合照学生识别功能（9分）

**原始要求**：
> 支持上传班级/活动大合照，后端对照片中的人脸进行批量识别，匹配人脸库中学生信息，生成匹配成功的学生名单，统计学生参与各类活动的频次，形成统计报表。

**数据流**：

```
上传合照 ──→ SCRFD 全检测（所有人脸框）
                │
                ├→ 逐人脸 ArcFace 提取 512d 特征
                ├→ 与数据库全部候选人余弦比对
                ├→ 去重（同一学生只计一次）
                ├→ 同步情绪分析
                ├→ 写入 AttendanceRecord + ActivityParticipation
                │
                └→ 返回 { 检测人脸数, 成功识别数, 结果列表 }
```

**需求分解**：

| 编号 | 需求 | 分值 | 说明 |
|---|---|---|---|
| F2.1 | 前端上传合照，后端批量识别人脸 | 3 | 拖拽/选择上传，预览后识别 |
| F2.2 | 准确率 ≥85%，处理 10-50 人，生成名单 | 3 | ArcFace 512d + 余弦相似度 + 阈值 0.58 |
| F2.3 | 自动记录每位学生活动参与次数 | 2 | ActivityParticipation 表，JOIN + GROUP BY |
| F2.4 | 生成统计报表（数字、表格、柱状图） | 1 | Dashboard 卡片 + 表格 + ECharts 柱状图 |

---

### F3 情绪分析功能（8分）

**原始要求**：
> 在考勤、合照识别过程中，同步对学生面部表情进行分析，并对情绪进行分类，可在前端查看统计结果。

**同步处理点**：

| 场景 | 触发位置 | 备注 |
|---|---|---|
| 考勤签到 | `attendance.py:88` — `emotion_service.analyze()` | 签到成功后记录 |
| 合照识别 | `group_photo.py:50` — `emotion_service.analyze()` | 每张识别成功的人脸 |
| 情绪记录 | `EmotionRecord` 表 — 含 student_id / timestamp / type / confidence / source | source 区分来源 |

**需求分解**：

| 编号 | 需求 | 分值 | 说明 |
|---|---|---|---|
| F3.1 | 同步提取面部特征 | 2 | 在考勤和合照流程中无需额外操作 |
| F3.2 | 完成情绪分类（7 类） | 3 | HSEmotion（主力）+ DeepFace（复核）+ OpenCV（保底） |
| F3.3 | 后端记录完整情绪数据 | 2 | EmotionRecord 表：学号/姓名/时间/类型/置信度/来源 |
| F3.4 | 前端查看统计结果 | 1 | ECharts 饼图 + 明细表格，点击可查学生详情 |

**7 类情绪**：happy / sad / angry / surprised / fearful / disgusted / neutral

**三引擎决策链**：
```
HSEmotion (EfficientNet-B0) → 置信度 ≥0.6 → 直接返回
        │ 不足
        ▼
DeepFace (EmotionNet) → 成功 → 返回
        │ 失败
        ▼
OpenCV 启发式 → smile cascade / eye cascade / Sobel 梯度
```

### F4 系统安全性能（18分）

**原始要求**：
> 优化识别算法，提升活体检测、人脸比对、合照识别的准确率，降低误识别率；实现系统异常处理。

**需求分解**：

| 编号 | 需求 | 分值 | 说明 |
|---|---|---|---|
| F4.1 | 活体检测抗照片伪造 | 7 | rPPG 一票否决 + DeepFace + LBP 纹理 + 摩尔纹 |
| F4.2 | 活体检测抗视频伪造 | 8 | 随机动作挑战 + rPPG + 摩尔纹 + 光流时序 |
| F4.3 | 实现教师/学生账号权限区分 | 3 | JWT + bcrypt + require_teacher 依赖注入 |

**攻击防御矩阵**：

| 攻击方式 | 拦截层 | 失效条件 |
|---|---|---|
| 打印照片 | rPPG（无脉搏）+ LBP（纸纹理）+ DeepFace | 高质量照片 + 真人站在旁边（rPPG 仍可穿透） |
| 手机/平板屏幕翻拍 | rPPG + 摩尔纹（干涉条纹）+ DeepFace | 极端高分辨率屏幕 |
| 视频重放（录别人） | rPPG + 摩尔纹 + **随机动作挑战** | 攻击者恰好录制了正确动作+高保真 |
| 手持照片晃动 | 光流时序（刚体平移检测） | 模拟真人的非刚体晃动 |
| 快速闪现照片 | 多帧分析（rPPG 需 30 帧以上） | 单帧提交绕过 |
| 过曝/过暗 | 动态阈值自动适应 | DeepFace 模型本身失效 |

**权限矩阵**：

| 操作 | 教师 | 学生 |
|---|---|---|
| 签到 | ✅ | ✅ |
| 查看考勤记录 | ✅（全部） | ✅（仅自己） |
| 增删改学生 / 导入人脸 | ✅ | ❌ |
| 合照识别 | ✅ | ❌ |
| 查看统计 | ✅ | ✅ |

## 三、非功能需求

| 编号 | 需求 | 指标 | 当前方案 |
|---|---|---|---|
| NF1 | 人脸识别置信度 | ≥ 58%（可配置） | ArcFace 512d 余弦相似度，`face_threshold=0.58` |
| NF2 | 合照识别准确率 | ≥ 85%，支持 10-50 人 | SCRFD + ArcFace，去重逻辑 |
| NF3 | 活体检测抗攻击 | 抵御打印照片/屏幕翻拍/视频重放 | 11 维评分 + rPPG 一票否决 |
| NF4 | 浏览器兼容 | Chrome / Firefox / Edge | `getUserMedia` API + Element Plus |
| NF5 | 人脸检测响应 | < 2 秒 | SCRFD-10GF CPU 推理 ~200ms |
| NF6 | 部署要求 | 单机可运行，无需 GPU | InsightFace CPU 模式, PyTorch CPU |
