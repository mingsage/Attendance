# Attendance 架构升级方案

## 改动总览

| 模块 | 当前 | 升级后 | 理由 |
|---|---|---|---|
| 人脸检测 | OpenCV YuNet | **InsightFace SCRFD** | 更快、更准，自带 5 关键点 |
| 人脸识别 | OpenCV SFace (128d) | **InsightFace ArcFace (512d)** | NIST-FRVT 排名第一，512d 区分力更强 |
| 情绪分析 | DeepFace → OpenCV 回退 | **DeepFace + HSEmotion 混合** | 双引擎互备，DeepFace 鲁棒 + HSEmotion SOTA |
| 反欺骗 | 7 维单帧评分 | **11 维多帧评分** | 新增 rPPG/摩尔纹/光流/动态阈值 |

### 1.2 InsightFace 配置

```python
from insightface.app import FaceAnalysis

class FaceService:
    def __init__(self):
        self.app = FaceAnalysis(
            name='buffalo_l',           # SCRFD-10GF + ResNet50@WebFace600K
            providers=['CPUExecutionProvider'],
            allowed_modules=['detection', 'recognition']  # 只加载检测+识别
        )
        self.app.prepare(ctx_id=-1, det_size=(640, 640), det_thresh=0.5)

    def detect_faces(self, image):
        faces = self.app.get(image, max_num=1)  # 考勤只取最大人脸
        if not faces:
            return []
        return [(f.bbox, f.det_score, f.kps) for f in faces]

    def extract_feature(self, image, face=None):
        if face:
            return face.normed_embedding  # 512d, L2 已归一化
        faces = self.app.get(image, max_num=1)
        return faces[0].normed_embedding if faces else None
```

## 二、情绪分析 → DeepFace + HSEmotion 混合

### 2.1 混合策略

```
人脸裁剪图
    │
    ▼
┌─────────────────────┐
│  HSEmotion           │  ← 主引擎（轻量、快、SOTA）
│  EfficientNet-B0     │
│  8 类情绪            │
└─────────┬───────────┘
          │ confidence ≥ 0.6 ?
          ├── YES → 直接返回
          └── NO  → 进入 DeepFace
                      │
                      ▼
              ┌──────────────┐
              │  DeepFace     │  ← 复核引擎（鲁棒、成熟）
              │  EmotionNet   │
              │  7 类情绪     │
              └──────┬───────┘
                     │ 仍失败 ?
                     └──→ OpenCV 启发式回退（保底）
```

### 2.2 核心代码

```python
class EmotionService:
    def __init__(self):
        self.hsemotion = HSEmotionRecognizer(
            model_name='enet_b0_8_best_afew', device='cpu')

    def analyze(self, image, box=None):
        face = self._crop_face(image, box)

        # 主引擎：HSEmotion
        try:
            label, scores = self.hsemotion.predict_emotions(face, logits=False)
            max_conf = float(scores.max())
            if max_conf >= 0.6:
                return self._map_label(label), round(max_conf, 3)
        except Exception:
            pass

        # 复核引擎：DeepFace
        try:
            from deepface import DeepFace
            result = DeepFace.analyze(face, actions=["emotion"],
                                       enforce_detection=False, silent=True)
            if isinstance(result, list):
                result = result[0]
            emotion = result["dominant_emotion"]
            confidence = round(result["emotion"][emotion] / 100.0, 3)
            return emotion, confidence
        except Exception:
            pass

        # 保底：OpenCV 启发式
        return self._fallback_analyze(face)

    def _map_label(self, hs_label):
        """8 类 → 7 类映射"""
        mapping = {
            'Happiness': 'happy', 'Sadness': 'sad', 'Anger': 'angry',
            'Surprise': 'surprised', 'Fear': 'fearful', 'Disgust': 'disgusted',
            'Contempt': 'disgusted', 'Neutral': 'neutral',
        }
        return mapping.get(hs_label, 'neutral')
```

### 2.3 优势

- HSEmotion 主引擎：PyTorch、轻量、SOTA 准确率，首次推理 ~200ms
- DeepFace 复核：成熟稳定，处理 HSEmotion 置信度不足的边界情况
- OpenCV 回退：极端情况下仍可工作
- 三引擎互备，最大化准确率 + 鲁棒性

### 3.2 rPPG 脉搏波检测（维度 8，权重 15%）

**原理**：心脏跳动导致面部毛细血管血量周期性变化 → 皮肤反射率微小波动 → 摄像头像素的绿色通道（G channel）可捕捉。照片/视频无此生物节律。

**实现**：

```python
class rPPGDetector:
    """远程光电容积脉搏波：从连续帧的面部 ROI 中提取心率信号。"""

    def __init__(self):
        self.signal_buffer = []      # 最近 N 帧的 G 通道均值
        self.buffer_size = 60        # 2秒 × 30fps（实际取考勤的 1-2 秒）
        self.fs = 30.0               # 采样率（近似）
        self.low_freq = 0.7          # 心率下限 Hz（≈42 bpm）
        self.high_freq = 3.0         # 心率上限 Hz（≈180 bpm）

    def add_frame(self, face_roi):
        """从裁剪的人脸区域提取 G 通道均值。"""
        g_channel = face_roi[:, :, 1].astype(np.float32)  # BGR → G=索引1
        self.signal_buffer.append(float(np.mean(g_channel)))
        if len(self.signal_buffer) > self.buffer_size:
            self.signal_buffer.pop(0)

    def compute_score(self):
        """FFT → 带通滤波 → 检查是否在心率频段检测到显著峰值。"""
        if len(self.signal_buffer) < 30:
            return 0.0, "rPPG 采样不足"

        signal = np.array(self.signal_buffer)
        signal = signal - np.mean(signal)

        # FFT
        n = len(signal)
        fft = np.abs(np.fft.rfft(signal))
        freqs = np.fft.rfftfreq(n, 1.0 / self.fs)

        # 带通 0.7-3.0 Hz
        mask = (freqs >= self.low_freq) & (freqs <= self.high_freq)
        if not mask.any():
            return 0.0, "rPPG 无心率频段信号"

        band_power = fft[mask]
        noise_power = fft[~mask] if (~mask).any() else np.array([1.0])

        peak_idx = np.argmax(band_power)
        peak_freq = freqs[mask][peak_idx]
        peak_ratio = band_power[peak_idx] / (np.mean(noise_power) + 1e-6)

        # 评分：峰值明显 → 真人
        if peak_ratio > 3.0:
            return min(peak_ratio / 8.0, 1.0), f"rPPG 检测到脉搏信号 ({peak_freq*60:.0f} bpm)"
        elif peak_ratio > 1.5:
            return 0.3, "rPPG 信号较弱"
        return 0.0, "rPPG 未检测到脉搏，疑似照片/屏幕"
```

**关键点**：
- 只取绿色通道（G）——血红蛋白对绿光吸收最敏感
- 对光照变化敏感，需先做去趋势（detrend）处理
- 可作为**一票否决项**：rPPG 得分 < 0.15 → 直接判为欺骗，不需要看其他维度

### 3.3 摩尔纹专项检测（维度 9，权重 10%）

**原理**：翻拍电子屏幕时，摄像头像素网格与屏幕像素网格叠加产生干涉条纹（莫尔纹）。与通用 FFT 不同，摩尔纹检测找的是**特定方向的周期性尖峰**，而非通用频谱分布。

**实现**：

```python
class MoireDetector:
    """专门检测翻拍屏幕时产生的莫尔纹。"""

    def detect(self, gray):
        h, w = gray.shape
        # 2D FFT
        fft = np.fft.fft2(gray.astype(np.float32))
        fft_shift = np.fft.fftshift(fft)
        magnitude = np.abs(fft_shift) + 1e-6

        cy, cx = h // 2, w // 2
        max_r = min(h, w) // 2
        mid_start = int(max_r * 0.15)   # 排除直流分量
        mid_end = int(max_r * 0.60)

        # 在中高频区域内搜索局部峰值
        Y, X = np.ogrid[:h, :w]
        dist = np.sqrt((Y - cy)**2 + (X - cx)**2)
        mask = (dist >= mid_start) & (dist <= mid_end)
        masked_mag = magnitude * mask

        # 找异常峰值（可能是摩尔纹）
        mean_mag = float(np.mean(masked_mag[mask]))
        std_mag = float(np.std(masked_mag[mask])) if mean_mag > 0 else 1.0

        # 超过均值 4σ 的峰值视为摩尔纹
        peak_mask = (masked_mag > mean_mag + 4 * std_mag) & mask
        peak_count = int(np.sum(peak_mask))

        if peak_count > 50:
            return 0.0, f"检测到大量莫尔纹峰值({peak_count})，疑似屏幕翻拍"
        elif peak_count > 10:
            return 0.3, f"检测到异常频率峰值({peak_count})"
        return 1.0, "未检测到莫尔纹"
```

**与通用 FFT 的区别**：

| | 通用 FFT（维度 2） | 摩尔纹专项（维度 9） |
|---|---|---|
| 分析方法 | 径向能量分布 | 局部峰值搜索 |
| 目标 | 面部的自然频谱衰减 | 屏幕+摄像头叠加产生的异常尖峰 |
| 对背景干扰的敏感性 | 中等（径向平均会掩盖局部尖峰） | 低（只看异常离群点） |

### 3.4 时序一致性分析（维度 10，权重 12%）

**原理**：手持照片晃动时，面部区域整体平移，内部器官无相对运动。真人头部运动时，面部器官（眼睛、嘴角）会产生**独立的微运动**，且与背景运动模式不同。

**实现**：光流法 + 运动一致性分析。

```python
class TemporalConsistency:
    """多帧运动一致性分析，区分真人 vs 照片晃动。"""

    def __init__(self):
        self.prev_gray = None
        self.prev_keypoints = None  # 面部关键点

    def update(self, gray, keypoints):
        """每帧更新并计算运动向量一致性。"""
        if self.prev_gray is None:
            self.prev_gray = gray
            self.prev_keypoints = keypoints
            return 0.5, "时序分析初始化中"

        # 1. 稠密光流 — 整脸区域的运动向量
        flow = cv2.calcOpticalFlowFarneback(
            self.prev_gray, gray, None,
            0.5, 3, 15, 3, 5, 1.2, 0)

        # 2. 提取面部中心区域和外围区域的光流
        h, w = gray.shape
        cy, cx = h // 2, w // 2
        r = min(h, w) // 4
        center_flow = flow[cy-r:cy+r, cx-r:cx+r]
        outer_flow = flow

        # 3. 计算运动向量的一致性（方差越小 → 越像刚体平移 → 越像照片）
        center_std = float(np.std(center_flow))
        outer_std = float(np.std(outer_flow))
        flow_diff = float(np.mean(np.abs(
            center_flow.mean(axis=(0,1)) - outer_flow.mean(axis=(0,1))
        )))

        # 4. 关键点相对位移 — 眼睛/嘴角是否有独立微运动
        kp_dispersion = 0.0
        if self.prev_keypoints is not None and keypoints is not None:
            if len(keypoints) >= 5 and len(self.prev_keypoints) >= 5:
                kp_shifts = np.linalg.norm(
                    keypoints[:5] - self.prev_keypoints[:5], axis=1)
                kp_dispersion = float(np.std(kp_shifts))

        self.prev_gray = gray
        self.prev_keypoints = keypoints

        # 5. 评分
        score = 0.5

        # 光流方差大 → 面部内部有独立运动 → 真人
        if center_std > 1.5:
            score += 0.3

        # 关键点分散度高 → 器官相对运动 → 真人
        if kp_dispersion > 0.8:
            score += 0.2
        elif kp_dispersion < 0.2 and center_std < 0.5:
            score = 0.1  # 高度一致的刚体运动 → 照片

        if score < 0.3:
            return score, "面部运动异常一致，疑似照片晃动"
        return min(score, 1.0), "时序一致性正常"
```

**攻击类型识别**：

| 场景 | 光流模式 | 检测结果 |
|---|---|---|
| 真人自然晃动 | 面部内部运动多样，关键点相对位移分散 | 正常通过 |
| 手持照片晃动 | 全脸刚体平移，关键点间位移一致 | 标记异常 |
| 视频重放（人脸在动） | 光流正常，但配合 rPPG + 摩尔纹检测 | 被其他维度拦截 |
| 完全静止 | 光流接近零 | 正常（可能只是用户很稳），rPPG 仍可工作 |

### 3.5 动态阈值（维度 11）

**原理**：光照和清晰度影响所有传统 CV 维度的可靠性。0.55 的固定阈值在过曝/过暗场景下过于严苛或过于宽松。

**实现**：

```python
def compute_dynamic_threshold(base_threshold, brightness, blur_score):
    """根据光照和清晰度动态调整判定阈值。

    base_threshold: 0.55（标准实验室阈值）
    brightness: 灰度均值（理想值 ~120）
    blur_score: Laplacian 方差归一化得分
    """
    brightness_error = abs(brightness - 120) / 120.0  # 0~1
    blur_penalty = max(0, 1.0 - blur_score)            # 0~1

    # 过暗/过亮 → 传统维度不可靠 → 提高阈值（更严格）
    alpha = 0.08  # 亮度修正系数
    beta = 0.05   # 模糊修正系数

    if brightness < 40 or brightness > 200:
        # 极端光照 → 大幅提高阈值
        alpha = 0.15

    adjusted = base_threshold + alpha * brightness_error + beta * blur_penalty
    return min(adjusted, 0.75)  # 上限 0.75，留给 DeepFace 决策空间
```

**公式含义**：
- 画面过暗（brightness < 40）：传统 CV 维度噪声大 → 阈值升到 0.60-0.65 → 更依赖 DeepFace
- 画面过亮（brightness > 200）：细节丢失 → 阈值同样提升
- 画面模糊（blur_score 低）：同样提高阈值
- 标准光照 + 清晰：保持 0.55 基准

### 3.6 综合评分流程

```python
def analyze(self, image, frames_buffer=None, keypoints_buffer=None):
    """11 维综合活体检测。"""

    gray = cv2.cvtColor(image, cv2.COLOR_BGR2GRAY)

    # ── 传统 7 维 ──
    deepface_score  = self._deepface_spoof_score(image)     # 40%
    freq_score      = self._frequency_score(gray)            # 5%
    lbp_score       = self._lbp_score(gray)                  # 5%
    color_score     = self._color_diversity_score(image)     # 3%
    blur_score      = self._blur_score(gray)                 # 2%
    brightness      = float(np.mean(gray))                   # (元数据)
    brightness_score = self._brightness_score(brightness)    # 2%
    texture_score   = self._texture_score(gray)              # 2%

    # ── 新增 3 维 ──
    rppg_score      = self.rppg.compute_score()              # 15%
    moire_score     = self.moire.detect(gray)                # 10%
    temporal_score  = self.temporal.update(gray, keypoints)  # 12%

    # ── 加权融合 ──
    if deepface_score is not None:
        score = (
            0.40 * deepface_score
            + 0.15 * rppg_score
            + 0.12 * temporal_score
            + 0.10 * moire_score
            + 0.05 * freq_score
            + 0.05 * lbp_score
            + 0.03 * color_score
            + 0.02 * blur_score
            + 0.02 * brightness_score
            + 0.02 * texture_score
        )
        # rPPG 一票否决
        if rppg_score < 0.15:
            return score, "活体检测失败：rPPG 未检测到脉搏信号，疑似照片/屏幕"
    else:
        # DeepFace 不可用时的退化方案
        score = (
            0.25 * rppg_score
            + 0.20 * temporal_score
            + 0.15 * moire_score
            + 0.15 * freq_score
            + 0.12 * lbp_score
            + 0.08 * color_score
            + 0.03 * blur_score
            + 0.02 * brightness_score
            + 0.02 * texture_score
        ) / 0.92
        # 无 DeepFace 时 rPPG 一票否决阈值更严
        if rppg_score < 0.10:
            return score, "活体检测失败：rPPG 未检测到脉搏信号，疑似照片/屏幕"

    # ── 动态阈值 ──
    threshold = self._dynamic_threshold(0.55, brightness, blur_score)

    if score < threshold:
        # 错误原因优先级排序
        if deepface_score is not None and deepface_score < 0.30:
            return score, "活体检测失败：疑似照片/屏幕翻拍"
        if moire_score < 0.15:
            return score, "活体检测失败：检测到屏幕翻拍痕迹"
        if rppg_score < 0.25:
            return score, "活体检测失败：未检测到生物脉搏信号"
        return score, f"活体检测失败：综合得分{score:.2f}低于阈值{threshold:.2f}"

    return score, "活体检测通过"
```

### 3.7 新攻击场景覆盖

| 攻击方式 | 旧方案（7维单帧） | 新方案（11维多帧） |
|---|---|---|
| 打印照片 | DeepFace/LBP 可能检出 | rPPG 直接判死（无脉搏） |
| 高清平板翻拍 | 通用 FFT 可能漏 | 摩尔纹专项 + rPPG 双重拦截 |

| 快速闪现照片 | 单帧盲区 | 多帧关联，至少需要 N 帧有效 |
| 强光/暗光环境 | 传统维度失真 | 动态阈值自动适配 |

## 五、风险矩阵

| 风险 | 概率 | 影响 | 缓解措施 |
|---|---|---|---|
| InsightFace buffalo_l 下载失败 | 中 | 检测+识别不可用 | 保留 OpenCV YuNet 回退分支 |
| rPPG 对深色肤色敏感度降低 | 中 | 脉搏检测偏差 | 只设 15% 权重，不一票否决，配合 DeepFace |
| 光流计算开销影响实时性 | 低 | 连续捕捉 800ms 间隔不够 | 降分辨率计算（320×240），GPU 加速 |
| HSEmotion timm 版本冲突 | 低 | 情绪分析不可用 | 回退到 DeepFace + OpenCV |
| 多帧采集期间用户移动 | 中 | 光流/rPPG 信号质量差 | 前端做稳定提示；异常帧自动丢弃 |
