# 第三方库选型分析：DeepFace / InsightFace / HSEmotion

> 目标：评估三者在 Attendance 项目中的角色分工，确定最优技术组合。

---

## 一、当前项目依赖现状

Attendance 项目的人脸相关能力由以下组件拼合而成：

| 能力 | 当前方案 | 依赖的库 | 是否自研 |
|---|---|---|---|
| 人脸检测 | OpenCV YuNet (ONNX) | `opencv-python` | 自研管线 |
| 人脸识别 | OpenCV SFace (ONNX) | `opencv-python` | 自研管线 |
| 活体反欺骗 | DeepFace MiniFASNet | `deepface`（**可选依赖，未写入 requirements.txt**） | 调库 |
| 情绪分析 | DeepFace EmotionNet → OpenCV 启发式回退 | `deepface`（同上）+ `opencv-python` | 调库 + 自研回退 |

结论：**DeepFace 只在两处被使用——反欺骗和情绪分析**。人脸检测和识别完全独立于 DeepFace。

## 二、InsightFace 调研

### 2.1 来源

- GitHub 官方仓库：`D:\code\program\neirong\insightface`
- PyPI 包：`insightface==0.7.3`
- 许可证：MIT（代码），模型仅限非商业研究

### 2.2 提供的能力

| 能力 | 模型 | 质量评估 |
|---|---|---|
| 人脸检测 | SCRFD / RetinaFace | **业界领先**，速度快、精度高、支持 5 关键点 |
| 人脸识别 | ArcFace (ResNet50/100) | **SOTA 级别**，512 维嵌入，NIST-FRVT 排名第一 |
| 人脸对齐 | 106 点 2D + 68 点 3D 关键点 | 远超 DeepFace 的基础对齐 |
| 性别/年龄 | Attribute 模型 | 可用，非本项目核心需求 |
| 反欺骗 | **❌ 不存在于开源 Python 包** | InspireFace C++ SDK 有，但那是独立的商业产品 |
| 情绪 | **❌ 不存在** | 无任何情绪分析能力 |

### 2.3 反欺骗的真相

InsightFace README 提到 "CVPR23 FAS-Wild Challenge"（人脸反欺骗挑战赛），但仓库中：
- `challenges/cvpr23-fas-wild/` 只有比赛说明和数据集链接
- 没有预训练的反欺骗模型权重
- 没有推理代码
- Python 包中搜索 `spoof`、`anti_spoofing`、`liveness` 关键词——**零匹配**

InsightFace 的反欺骗能力属于 **InspireFace**，一个独立的 C++ 商业 SDK（`cpp-package/inspireface/`）。它与 `pip install insightface` 完全无关。

### 2.4 典型调用方式

```python
from insightface.app import FaceAnalysis

app = FaceAnalysis(providers=['CPUExecutionProvider'])
app.prepare(ctx_id=-1, det_size=(640, 640))  # ctx_id=-1 = CPU

img = cv2.imread("photo.jpg")
faces = app.get(img)  # 自动运行检测→识别→对齐→属性

for face in faces:
    face.bbox              # [x1, y1, x2, y2]
    face.det_score         # 检测置信度
    face.kps               # 5 个关键点
    face.normed_embedding  # 512 维 L2 归一化嵌入
    face.landmark_2d_106   # 106 个 2D 关键点
    face.pose              # [pitch, yaw, roll]
    face.gender / face.age # 性别/年龄
```

### 2.5 对本项目的价值

| 维度 | 评价 |
|---|---|
| 替换 DeepFace | **不行**——InsightFace 不提供反欺骗和情绪 |
| 替换 OpenCV YuNet+SFace | **技术上有优势**——SCRFD 比 YuNet 快且准，ArcFace 比 SFace 强。但需要改写整个 `face_service.py`，工作量大 |
| 增强动作验证 | **有价值**——106 点关键点的 `pose`（pitch/yaw/roll）可以更精确地判断转头/张嘴等动作，比 YuNet 的 15 维向量精度高 |

### 2.6 推荐使用方式

**不全面替换，仅作为动作验证增强**。InsightFace 的 106 点关键点 + 3D 姿态估计可以大幅提升 `verify_action()` 的精度：

- `face.pose[1]`（yaw）直接给出转头角度，不需要手工算鼻尖偏移比例
- 106 点覆盖嘴部轮廓，张嘴/微笑判断更可靠

## 三、HSEmotion 调研

### 3.1 来源

- 本地副本：`D:\code\program\neirong\hsemotion`
- PyPI 包：`hsemotion==0.3.0`
- 论文：[Classifying Emotions and Engagement in Online Learning](https://paperswithcode.com/sota/facial-expression-recognition-on-affectnet)
- 许可证：Apache 2.0（学术+商业无限制）
- AffectNet 数据集上 SOTA

### 3.2 核心实现

**只有 1 个源文件**（`hsemotion/facial_emotions.py`，122 行），极度精简：

```python
from hsemotion.facial_emotions import HSEmotionRecognizer

fer = HSEmotionRecognizer(model_name='enet_b0_8_best_afew', device='cpu')
emotion, scores = fer.predict_emotions(face_img, logits=True)
```

### 3.3 可用模型

| 模型名 | 骨干网络 | 输入尺寸 | 类别数 | 特点 |
|---|---|---|---|---|
| `enet_b0_8_best_vgaf` | EfficientNet-B0 | 224×224 | 8 类（含 Contempt） | 轻量、快速，默认 |
| `enet_b0_8_best_afew` | EfficientNet-B0 | 224×224 | 8 类 | 不同训练集 |
| `enet_b0_8_va_mtl` | EfficientNet-B0 | 224×224 | 8 类 + Valence/Arousal | 多任务 |
| `enet_b2_8` | EfficientNet-B2 | 260×260 | 8 类 | 更大、更准 |
| `enet_b2_7` | EfficientNet-B2 | 260×260 | 7 类（无 Contempt） | 基础情绪集 |

模型自动从 GitHub 下载到 `~/.hsemotion/`，无需手动管理。EfficientNet-B0 约 16-20 MB，B2 约 25-30 MB。

### 3.4 情绪标签

8 类：`Anger, Contempt, Disgust, Fear, Happiness, Neutral, Sadness, Surprise`

7 类：`Anger, Disgust, Fear, Happiness, Neutral, Sadness, Surprise`

与 DeepFace 的 7 类（happy/sad/angry/surprised/fearful/disgusted/neutral）对应关系：

| DeepFace | HSEmotion | 备注 |
|---|---|---|
| happy | Happiness | 直接对应 |
| sad | Sadness | 直接对应 |
| angry | Anger | 直接对应 |
| surprised | Surprise | 直接对应 |
| fearful | Fear | 直接对应 |
| disgusted | Disgust | 直接对应 |
| neutral | Neutral | 直接对应 |
| — | Contempt | DeepFace 没有，HSEmotion 8类模型独有 |

### 3.5 依赖项

```
numpy, pillow, torch, torchvision, timm==0.9.*
```

**关键优势**：纯 PyTorch 生态。不依赖 TensorFlow/Keras，不与 DeepFace 冲突。

### 3.6 与 DeepFace 情绪识别对比

| 维度 | DeepFace | HSEmotion |
|---|---|---|
| 模型 | EmotionNet（基于 VGG） | EfficientNet-B0/B2 |
| 框架 | TensorFlow / Keras | PyTorch |
| 模型大小 | ~100+ MB（含 TF 运行时） | ~20 MB（仅 .pt 文件） |
| 推理速度 | 较慢（VGG 较深） | 快（EfficientNet 高效） |
| 准确率 | 中等 | AffectNet SOTA |
| 许可证 | MIT | Apache 2.0 |
| 依赖数量 | 多（TF + 多个子依赖） | 少（torch + timm） |
| 内置人脸检测 | 有（但本项目不用） | 无（需要预先裁剪人脸） |
| 输出格式 | 主导情绪 + 百分比 | 主导情绪 + logits/scores |
| 特征提取 | 不支持 | 支持 1280 维嵌入 |
| 社区成熟度 | 高（GitHub 40k+ stars） | 低（较新） |

### 3.7 对本项目的价值

**可以作为 DeepFace 情绪识别的轻量化替代**。

当前 `emotion_service.py` 的结构是：
```python
try:
    result = DeepFace.analyze(face, actions=["emotion"], ...)  # TF 后端，慢
except:
    return self._fallback_analyze(face)  # OpenCV 启发式，不准
```

替换为 HSEmotion：
```python
try:
    emotion, scores = hsemotion_fer.predict_emotions(face, logits=False)  # PyTorch，快
except:
    return self._fallback_analyze(face)  # 仍保留回退
```

好处：
1. 去掉 DeepFace 的情绪依赖，不再需要 TF/Keras 用于情绪
2. 推理速度大幅提升（EfficientNet-B0 vs VGG）
3. 1280 维特征可存储用于后续分析
4. 8 类模型比 DeepFace 多 Contempt 类别

注意事项：
1. **HSEmotion 不做人脸检测**——需要先用人脸框裁剪出面部区域再传入。当前项目已有 `_crop_face()` 方法，正好对接口
2. **标签需要映射**——`Happiness` → `happy`、`Anger` → `angry` 等，保持前端兼容
3. `timm==0.9.*` 版本约束需要关注

## 五、推荐方案

### 5.1 三库分工

```
┌─────────────────────────────────────────────────────┐
│                 Attendance 人脸管线                   │
├───────────────┬──────────────┬──────────────────────┤
│  人脸检测+识别  │  活体反欺骗   │      情绪分析         │
│  (保持现有)    │  (DeepFace)  │    (HSEmotion 替换)   │
├───────────────┼──────────────┼──────────────────────┤
│ OpenCV YuNet  │ MiniFASNet   │ EfficientNet-B0       │
│ OpenCV SFace  │ + FFT 频域   │ 8类情绪               │
│ (已验证稳定)   │ + LBP 纹理   │ 轻量、快速、SOTA       │
│               │ + 色彩分析   │                       │
│               │ + 动作验证   │                       │
└───────────────┴──────────────┴──────────────────────┘
         ↑                              ↑
    InsightFace                   完全移除 TF/Keras
   (暂不引入，若需要               依赖，仅需 PyTorch
    增强动作验证可引入)
```

### 5.2 具体改动

| 步骤 | 文件 | 改动 |
|---|---|---|
| 1 | `requirements.txt` | 添加 `hsemotion`、保留 `deepface`（仅用于反欺骗）；不再需要为情绪引入额外依赖 |
| 2 | `emotion_service.py` | `analyze()` 改为：优先 HSEmotion EfficientNet-B0（`enet_b0_8_best_afew`），失败回退 OpenCV 启发式；移除 DeepFace 情绪调用 |
| 3 | `emotion_service.py` | 新增标签映射：`Happiness→happy`, `Sadness→sad`, `Anger→angry`, `Surprise→surprised`, `Fear→fearful`, `Disgust→disgusted`, `Neutral→neutral`, `Contempt→disgusted` |
| 4 | `liveness_service.py` | 保持 DeepFace 用于 MiniFASNet 反欺骗；提高阈值（0.38→0.55） |
| 5 | `frontend` | 不变——情绪标签映射在后端完成，前端仍用 `happy/sad/angry/surprised/fearful/disgusted/neutral` |

### 5.3 为什么不全换 InsightFace

| 方案 | 问题 |
|---|---|
| InsightFace 替换 YuNet+SFace | OpenCV YuNet+SFace 已验证可工作、够用。换引擎的收益（检测/识别精度提升）对考勤场景不是瓶颈——瓶颈在活体和情绪，这正是 InsightFace 不提供的 |
| 先引入 InsightFace 做动作验证 | 有价值但工作量中等（需加载 InsightFace 模型包 326MB），可以在活体检测基线达标后再引入 |
| **当下优先级** | **先把反欺骗（DeepFace）+ 情绪（HSEmotion）做扎实，活体检测能过 10 分再说** |

### 5.4 最终依赖清单

```
# 反欺骗（唯一需要 DeepFace 的地方）
deepface>=0.0.79

# 情绪识别（替换 DeepFace 的情绪功能）
hsemotion>=0.3.0

# 人脸检测+识别（保持不变）
opencv-python>=4.12.0.88
numpy>=2.0

# 频域/纹理分析（保持不变）
scikit-image

# 移除的依赖
# - deepface 的情绪功能（代码层面移除调用，但 deepface 包仍保留）
# - 不再需要 tensorflow/keras 作为情绪推理引擎（hsemotion 用 PyTorch）
```

## 六、风险与对策

| 风险 | 影响 | 对策 |
|---|---|---|
| HSEmotion 较新、社区小 | 遇到 bug 难排查 | 保留 OpenCV 启发式回退，HSEmotion 失败时自动降级 |
| `timm==0.9.*` 版本锁定 | 未来可能与其他包冲突 | 使用虚拟环境隔离，或在后续版本中跟进 |
| DeepFace + tensorflow 依赖仍存在 | 包体积大、安装慢 | 仅用于 MiniFASNet（推理很快），是当前最可靠的反欺骗选项 |
| InsightFace 若后续引入会增大模型体积 | 部署包变大 | 按需加载，动作验证非必须不加载 InsightFace |
