# 班级考勤系统 — 课程设计报告

> **项目名称**：基于人脸识别的课堂考勤系统  
> **技术栈**：Vue 3 + Element Plus（前端） / FastAPI + SQLAlchemy（后端） / InsightFace + DeepFace + HSEmotion（AI）  
> **数据库**：SQLite  
> **分支**：`feature/upgrade`

---

## 第一章 需求分析

### 1.1 项目背景

传统课堂考勤方式（点名、刷卡）存在效率低、代签等问题。本系统利用人脸识别技术实现非接触式快速考勤，并通过活体检测防止照片/视频欺骗，确保考勤数据真实可靠。

### 1.2 核心需求

| 序号 | 功能模块 | 核心需求 |
|---|---|---|
| 1 | 基础考勤 | 摄像头人脸采集 → 活体检测 → 人脸识别 → 考勤记录 |
| 2 | 合照识别 | 上传班级合照 → 批量检测识别人脸 → 生成参与名单 |
| 3 | 情绪分析 | 考勤/合照过程中同步提取面部特征 → 完成 7 类情绪分类 |
| 4 | 系统安全 | 活体检测抗照片/视频伪造攻击 → 教师/学生角色权限分离 |
| 5 | 数据管理 | 学生人脸库增删改查、批量导入 → 考勤记录查询/导出 Excel |

### 1.3 非功能需求

- 人脸识别置信度 ≥ 58%（可配置阈值）
- 合照识别支持 10-50 人，准确率 ≥ 85%
- 活体检测可有效抵御打印照片、屏幕翻拍、视频重放三种攻击
- 前端适配 Chrome/Firefox/Edge 主流浏览器

def get_current_user(token: str = Depends(oauth2_scheme),
                     db: Session = Depends(get_db)) -> User:
    username = decode_access_token(token)
    if not username:
        raise HTTPException(status_code=401, detail="登录已失效")
    return db.query(User).filter(User.username == username).first()
```

**实现**：这是系统的安全核心。采用 **11 维综合评分** 方案（`liveness_service.py`）：

**架构概览**：

```

    │

**维度 1：rPPG 脉搏波检测**（`liveness_service.py:11-80`）— **核心技术亮点**

原理：心脏跳动 → 面部毛细血管血量周期性变化 → 皮肤绿色通道反射率微小波动（肉眼不可见）。照片和视频无法模拟此生物信号。

```python
# liveness_service.py:46-58 — 信号累积 + FFT 分析
def add_frame(self, face_roi):
    g = face_roi[:, :, 1].astype(np.float32)   # 仅取绿色通道
    self.signal_buffer.append(float(np.mean(g))) # RO 均值即为 PPG 信号

def compute(self):
    signal = np.array(self.signal_buffer)
    signal = signal - np.mean(signal)
    # 去趋势 → FFT → 带通 0.7-3.0 Hz → 峰值检测
    fft = np.abs(np.fft.rfft(signal))
    mask = (freqs >= 0.7) & (freqs <= 3.0)
    peak_ratio = band_peak / noise_mean
    # 峰噪比 > 3.0 → 真人脉搏信号
```

**rPPG 一票否决规则**：得分 < 0.15 → 直接判为欺骗，不走后续判断。

**维度 2：摩尔纹专项检测**（`liveness_service.py:83-113`）

针对屏幕翻拍（手机/平板/电脑屏幕）的专属检测。与通用 FFT 径向分析不同，搜索局部异常尖峰：

```python
# liveness_service.py:103-110 — 局部峰值搜索
masked = magnitude * mask
peak_mask = (masked > mean_val + 4 * std_val) & mask  # 4σ 阈值
if peak_count > 50:        # 大量高频尖峰 = 屏幕翻拍的莫尔纹
    return 0.0, "摩尔纹峰值 {peak_count}，疑似屏幕翻拍"
```

**维度 3：光流时序一致性**（`liveness_service.py:116-172`）

区分"真人晃动"（器官有独立微运动）与"照片晃动"（整体刚体平移）：

```python
# liveness_service.py:147-156 — 运动一致性分析
flow = cv2.calcOpticalFlowFarneback(prev_gray, gray, ...)
center_std = float(np.std(center_flow))          # 中心区域光流方差
kp_dispersion = float(np.std(kp_shifts))          # 关键点相对位移分散度
if center_std > 1.5: score += 0.3                 # 高方差 = 真人
if kp_dispersion > 0.8: score += 0.2              # 关键点独立运动 = 真人
elif kp_dispersion < 0.2 and center_std < 0.5:
    score = 0.1  # 完美刚体运动 → 照片无疑
```

**维度 4：动态阈值**（`liveness_service.py:204-215`）

告别固定 0.55 阈值。过暗/过亮/模糊场景下传统 CV 维度不可靠，自动提高判定线：

```python
# liveness_service.py:204-215
def _dynamic_threshold(base, brightness, blur_score):
    brightness_error = abs(brightness - 120.0) / 120.0
    if brightness < 40 or brightness > 200:
        alpha = 0.15      # 极端光照 → 大幅提高
    adjusted = base + alpha * brightness_error + 0.05 * blur_penalty
    return min(adjusted, 0.75)   # 上限 0.75
```

##### 5.1.2 活体检测抗视频伪造攻击效果良好，杜绝虚假考勤（8分）

**五重防线**：

1. **随机动作挑战**：4 种动作中随机选 1 种，攻击者无法预知该播放哪种动作的视频
2. **rPPG 脉搏波**：视频压缩会破坏微弱的 PPG 信号
3. **摩尔纹专项**：摄像机拍电子屏幕必然产生干涉条纹
4. **光流时序分析**：视频重放的光流模式与真人自然运动有差异
5. **15 秒挑战过期**：`liveness-challenge` 接口返回 `expires_in: 15`

## 第四章 技术总结

### 核心指标汇总

| 指标 | 目标 | 实现方案 |
|---|---|---|
| 人脸检测 | 快速准确 | InsightFace SCRFD-10GF（320×320, CPU推理） |
| 人脸识别 | 置信度 ≥58% | InsightFace ArcFace ResNet50 512d |
| 活体检测 | 抗照片/视频/屏幕攻击 | 11 维综合：rPPG + 摩尔纹 + 光流 + DeepFace + 动态阈值 |
| 动作验证 | 4 种随机动作 | 微笑 Haar + 转头关键点偏移 + 张嘴比例 |
| 情绪分类 | 7 类 | HSEmotion（主力）+ DeepFace（复核）+ OpenCV（保底） |
| 合照识别 | 10-50 人 ≥85% | SCRFD 全检测 + ArcFace 逐人识别 + 去重 |
| 数据导出 | Excel 格式 | openpyxl 流式生成 |

### 创新点

1. **rPPG 脉搏波活体检测**：利用摄像头绿色通道捕捉心跳引起的皮肤颜色微变，从生物信号层面区分真人和照片/视频
2. **三引擎情绪混合架构**：HSEmotion SOTA 精度 + DeepFace 鲁棒性 + OpenCV 全天候保底
3. **动态阈值**：根据环境光照和清晰度自适应调整活体判定标准，解决不同教室/天气的泛化问题
4. **摩尔纹专项检测**：针对屏幕翻拍的 2D-FFT 局部峰值搜索，补充通用频域分析的盲区
